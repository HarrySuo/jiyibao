Three Differences

为什么它不是又一个
AI 会议纪要工具

01 / 主动，不等你催

不是你整理会议，
是系统先行动。

普通工具 录完以后等你想起来，再手动上传、再点总结、再回头找结果。
记忆宝 录音一发生，系统就开始工作；转写完成、纪要就绪、重点结论都会主动找人。
“主动触发 + 主动提醒”不是锦上添花，而是把工具变成产品的那条分界线。
02 / 编译，不只转写

不是给你一段文字，
是给你可用的理解结果。

普通工具 大多数产品停在转写和通用摘要层，很难真正进入经营判断。
记忆宝 按场景模板理解内容，把“客户拜访”“技术评审”“候选人面试”变成不同的价值结构。
模板的意义不在于格式，而在于把不同场景真正翻译成不同的经营信息。
03 / 积累，不是一次摘要

不是做完一份纪要，
而是长出一套企业记忆。

普通工具 每场会都是独立文件，每次追问都从头开始，经验没有复利。
记忆宝 所有沟通持续沉淀进同一套知识结构，历史上下文会越来越完整，组织记忆会越来越强。
最终买单逻辑，不是“省了几分钟写纪要”，而是“经验终于开始沉淀成资产”。

01 · 先理解它

用一句话理解记忆宝

记忆宝会把你的会议、录音、转写文本自动接入处理链路，完成转写、纪要整理、主动提醒，并继续做基于会议内容的知识编译与管理。

和普通纪要工具的区别 普通工具只是"整理完给你一份结果"。
记忆宝从采集开始持续工作：自动承接 → 自动编译 → 主动提醒 → 继续可追问 → 长期可管理。
最终留下的不是一份孤立纪要，而是可以持续积累和复用的个人知识资产。

08 · 常见问题

FAQ

为什么我收不到提醒？▾
按以下顺序检查：

1. 对记忆宝说 帮助，看有没有提示缺失 .env / 知识库路径 / Gateway / OPENCLAW_TARGET
2. 确认已执行过 install_openclaw.py install
3. 确认状态里有 ai-yunxin-jiyibao-auto-poller
4. 确认 OPENCLAW_TARGET 填的是你自己的目标
5. 注意：安装前已存在的老会议、老录音不会触发主动推送
网易会议秒录没有反应？▾
1. 确认在会议里手动打开了「秒录」（不是自动的）
2. 确认会议已经正常结束
3. 检查 .env 里 AUTO_ENABLE_NE_MEETING=true
4. 非网易员工需要补齐完整四项凭证
B100 没有反应？▾
1. 确认填的是你自己的 B100 账号密码，不是别人的
2. 确认录音已同步到服务端
3. 检查 .env 里 AUTO_ENABLE_B100=true
自检说 Gateway 不可达怎么办？▾
这通常说明 OpenClaw 自身没正常起来。先确认 OpenClaw 是否正常运行，再重新执行：
python3 scripts/auto/setup.py
非网易员工只填了 userUuid，可以吗？▾
不可以。非网易员工需要补齐完整四项：
NE_MEETING_APP_KEY / NE_MEETING_APP_SECRET / NE_MEETING_CORP_CODE / NE_MEETING_USER_UUID

网易员工但还没有会议账号的，可以先继续安装，系统会在首次触发秒录时自动创建账号。
我担心会沿用别人的账号配置？▾
clean-room 包已清除所有用户态字段：OPENCLAW_TARGET、NE_MEETING_USER_UUID、B100 账密、SMTP/POPO 提醒账号全部为空，不会沿用发包人的任何配置。
不想测 B100，可以先跳过吗？▾
可以。第一次体验完全可以只走路径 A（粘贴文字）和路径 B（网易会议秒录），B100 不是首次体验的必选项。
